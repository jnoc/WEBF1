Mitigating personal information exposure on the web

Introduction

Personal information is an important part of society. It is key that this information is in the right hands otherwise, through malicious intent it can be used against users (National Crime Agency, n.d.). Cybersecurity is critical when tackling these threats on the internet. The world is becoming increasingly digitized (Internet World Stats, 2017) leading to an acceleration of data being readily available to users. Thus, through mitigation of personal information, the threat to a user can be reduced. Mitigation is described as making something less harmful (Cambridge University Press, n.d.), hence preventing the information loss to start with is crucial as this will determine how freely the data would be leaked. Personal information after it has been leaked can be a difficult task for some users to comprehend. This is why there are ways in which a user can overcome this problem through its mitigation.

Prevention of personal information loss to begin with

By having a system put in place, security problems will not be down to the system itself yet will be down to the awareness of the users, as said by a notorious hacker “it's easier to manipulate people rather than technology” (K Mitnick, 2002). This system would involve the users to participate in testing and educated on how to protect their data all through an online learning system making it straightforward to understand (C Chen, R Shaw, S Yang, 2006). This can be implemented either in a company or household to deter users from leaking their company data or own personal data. Mitigation would involve users changing the way they act online, separate email accounts, phones and randomly created passwords would aid to their security within the internet. In office hours different levels of clearance would lessen the risk of accidental exposure due to specific personnel who understand the sensitivity of the data.

Other ways in which users can be cautious is understanding that the growing number of data collectors collect explicit information about anything a user is doing on the browser. Through the failure of security from 3rd party data collectors, cookies can be compared and linked to reused emails (B Krishnamurthy, K Naryshkin, C Wills, 2011). Therefore, it is key to understand that user actions can lead to vulnerability. Protection from this threat may include using a virtual private network (VPN) or an anonymous proxy to be routed through to the webpage. This will mask the user's location (Microsoft, n.d), yet further settings within the browser such as clearing your cookies will prevent tracking (Privacy, 2018). More privacy can be achieved by turning on ‘Do not track’ requests as well as installing an HTTPS plugin “that encrypts your communications with many major websites” (Electronic Frontier Foundation, n.d) when visiting HTTP pages.

Physical and poor network security may also lead to personal information loss. Through the use of USB or email, users with malicious intent can send files and export files to a physical copy to be handed over to an external user (P Gordon, 2007). To counter this, employers can have the security of the computers digitally and physically secured from these exploits that may be attempted by a user. A virtual desktop infrastructure (VDI) can be used to create “a walled garden” (P Gordon, 2007, pg 36) which would prevent the user from removing data via USB. While emails can be dealt with in an equivalent program that would be similar to blocking USB attempts.

Personal information loss and how can it be made less harmful

After data has been leaked it may seem hard to establish how to deal with that data being public. The first assessment of personal information loss is how sensitive is the data and how can it directly affect an individual. Through mitigation of personal information, an individual may look into if and how the type of information leaked requires a specific route to tackle the issue. This could involve criminal law, statute law, government regulation or common law (S Sproule, F Vachon, 2015). When looking into the depth of the personal information that has been leaked some factors should be considered, does the data impact your rights, can you take legal action or how high a risk is this in the hands of other people? From this, you can take the right action for protecting your leaked personal information.

Though personal information can be leaked by the user it is also possible that other users in contact with that user may accidentally leak personal data of that user without knowing it. Privacy settings on social media such as Facebook and Twitter allow users to hide sensitive and personal information (Facebook, 2018)(Twitter, 2018) from the ‘non-friends’ or ‘followers’, this may include a setting to hide pictures. However if another connected user posts a picture or birthday message for the private user but in the public domain, then that personal information has been leaked unknowingly. From this, it is important to understand how making sure that other users understand about individuals privacy. This can be easily removed by reminding that user or reporting it.

Conclusion

Mitigating personal information is key in preventing this personal information loss from being leaked in the first place. This is very important when thinking about the ways in which the attacks can occur, such as phishing, social engineering, file hijacking, keylogging, eavesdropping, and malware as these types of attacks have “increased almost fourfold between January 2016 and October 2017” (Cisco, n.d.). These can be fought against using some of the information found in this review, yet it is understandable that there are more ways to counter these attacks. As well as this, it is important to understand how users can approach their leaked information in the appropriate way.
